# CRM-система управления продавцами и транзакциями

REST API-сервис для управления данными о продавцах и их транзакциях.

## Функциональность

### Работа с продавцами
- CRUD операции.
- Хранение имени, контактной информации и даты регистрации продавца.

### Работа с транзакциями
- CRUD операции.
- Привязка транзакций к продавцам через внешний ключ.
- Поддержка трёх типов оплаты: CASH, CARD, TRANSFER.

### Аналитика
- Определение самого продуктивного продавца за выбранный период (день, месяц, квартал, год). Продуктивность определяется по наибольшей сумме транзакций за период.
- Получение списка продавцов с суммой транзакций ниже заданного порога.

### Обработка ошибок
- Централизованная обработка исключений с использованием GlobalExceptionHandler.
- Валидация входных данных с использованием Validation.
- Структурированные ответы об ошибках с кодами HTTP 400, 404, 500.

### Тестирование
- Юнит-тесты для бизнес-логики PeriodCalculator.
- Web-тесты для контроллеров.
- Интеграционные тесты с in-memory базой данных H2.
- Отчёт о покрытии кода через JaCoCo.

## Зависимости
- JDK 21+. [Инструкция по установке](https://habr.com/ru/companies/otus/articles/994150/)
- Docker & Docker Compose [Инструкция по установке](https://yandex.cloud/ru/docs/container-registry/operations/configure-docker)
- Gradle (или Gradle Wrapper) [Инструкция по установке](https://docs.gradle.org/current/userguide/installation.html)

## Быстрый старт
1. Устанавливаем все необходимые зависимости.
2. Запускаем PostgreSQL через Docker Compose: `docker-compose up -d`
    - База данных будет доступна на localhost:5432:
        - Имя БД: shift_postgres
        - Пользователь: admin
        - Пароль: admin
3. Сборка и запуск приложения: `./gradlew bootRun`
    - Приложение будет доступно http://localhost:8080.
4. Запуск тестов: `./gradlew test`
5. Запуск тестов с покрытием: `./gradlew test jacocoTestReport`
    - Сводка (index.html) будет доступна по `<Путь_до_корня_проекта>/build/jacocoHtml/index.html`

## API Документация
### Продавцы (`/api/sellers`)
#### GET /api/sellers
Возвращает список всех продавцов.

**Ответ:**

| Код | Описание                     |
|-----|------------------------------|
| 200 | Возвращает массив продавцов. |

#### GET /api/sellers/{id}
Возвращает информацию о конкретном продавце по ID.

**Параметры:**

| Параметр | Тип     | Обязательный | Описание                          |
|----------|---------|--------------|-----------------------------------|
| id       | Integer | да           | Уникальный идентификатор продавца |

**Ответ:**

| Код | Описание                          |
| --- | --------------------------------- |
| 200 | Возвращает данные продавца        |
| 404 | Продавец с указанным ID не найден |

**Пример ответа (200):**
```json
{
  "id": 1,
  "name": "Иванов Иван",
  "contactInfo": "ivan@example.com",
  "registrationDate": "2026-01-15T10:00:00"
}
```
**Пример ответа (404):**
```json
{
  "message": "Entity not found",
  "errorMessage": "Not found Seller by id = 999",
  "errorTime": "2026-05-15T10:30:00"
}
```
#### POST /api/sellers
Создаёт нового продавца

**Параметры:**

| Параметр    | Тип    | Обязательный | Описание                          |
| ----------- | ------ | ------------ | --------------------------------- |
| name        | String | да           | Имя продавца (не пустое)          |
| contactInfo | String | да           | Контактная информация (не пустая) |

**Ответ:**

| Код | Описание                                      |
| --- | --------------------------------------------- |
| 201 | Продавец успешно создан                       |
| 400 | Некорректные данные (пустое имя или контакт)  |

**Пример ответа (400):**
```json
{
  "message": "Bad request",
  "errorMessage": "Name is required",
  "errorTime": "2026-05-15T10:30:00"
}
```
#### PUT /api/sellers/{id}
Обновляет информацию о продавце

**Параметры:**

| Параметр    | Тип     | Обязательный | Описание                          |
| ----------- | ------- | ------------ | --------------------------------- |
| id          | Integer | да           | Уникальный идентификатор продавца |
| name        | String  | да           | Новое имя продавца                |
| contactInfo | String  | да           | Новая контактная информация       |

**Ответ:**

| Код | Описание                              |
| --- | ------------------------------------- |
| 200 | Продавец успешно обновлён             |
| 400 | Некорректные данные                   |
| 404 | Продавец с указанным ID не найден     |


#### DELETE /api/sellers/{id}
Удаляет продавца по ID

**Параметры:**

| Параметр | Тип     | Обязательный | Описание                          |
| -------- | ------- | ------------ | --------------------------------- |
| id       | Integer | да           | Уникальный идентификатор продавца |

**Ответ:**

| Код | Описание                                              |
| --- | ----------------------------------------------------- |
| 200 | Продавец успешно удалён                               |
| 404 | Продавец с указанным ID не найден                     |
| 400 | Нельзя удалить продавца с существующими транзакциями  |

**Пример ответа (400):**
```json
{
  "message": "Bad request",
  "errorMessage": "Cannot delete seller with existing transactions",
  "errorTime": "2026-05-15T10:30:00"
}
```
#### GET /api/sellers/analytics/top
Возвращает самого продуктивного продавца за указанный период

**Параметры:**

| Параметр | Тип       | Обязательный | Описание                                                 |
| -------- | --------- | ------------ | -------------------------------------------------------- |
| period   | String    | нет          | Период: day, month, quarter, year. По умолчанию day      |
| date     | LocalDate | нет          | Конкретная дата в формате YYYY-MM-DD. По умолчанию сегодня |

**Ответ:**

| Код | Описание                                         |
| --- | ------------------------------------------------ |
| 200 | Возвращает данные лучшего продавца               |
| 400 | Некорректный период                              |
| 404 | Нет данных за указанный период                   |

**Пример запроса:**
```
GET /api/sellers/analytics/top?period=month&date=2026-05-01
```

**Пример ответа (200):**
```json
{
  "id": 3,
  "name": "Сидоров Сидор",
  "contactInfo": "sidor@example.com",
  "registrationDate": "2026-01-10T09:00:00"
}
```

**Пример ответа (404):**
```json
{
  "message": "Entity not found",
  "errorMessage": "No top seller found for period",
  "errorTime": "2026-05-15T10:30:00"
}
```
#### GET /api/sellers/analytics/low-performers
Возвращает список продавцов с суммой транзакций меньше указанного порога

**Параметры:**

| Параметр  | Тип        | Обязательный | Описание                                                 |
| --------- | ---------- | ------------ | -------------------------------------------------------- |
| threshold | BigDecimal | да           | Порог суммы (продавцы с суммой меньше этого значения)    |
| period    | String     | нет          | Период: day, month, quarter, year. По умолчанию day      |
| date      | LocalDate  | нет          | Конкретная дата в формате YYYY-MM-DD. По умолчанию сегодня |

**Ответ:**

| Код | Описание                                         |
| --- | ------------------------------------------------ |
| 200 | Возвращает массив продавцов (может быть пустым)  |
| 400 | Не указан обязательный параметр threshold        |


### Транзакции (`/api/transactions`)
#### GET /api/transactions
Возвращает список всех транзакций

**Ответ:**

| Код | Описание                        |
| --- | ------------------------------- |
| 200 | Возвращает массив транзакций    |

**Пример ответа (200):**
```json
[
  {
    "id": 1,
    "sellerId": 1,
    "sellerName": "Иванов Иван",
    "amount": 1500.50,
    "paymentType": "CARD",
    "transactionDate": "2026-05-15T14:30:00"
  },
  {
    "id": 2,
    "sellerId": 2,
    "sellerName": "Петров Петр",
    "amount": 3200.00,
    "paymentType": "CASH",
    "transactionDate": "2026-05-16T10:15:00"
  }
]
```
#### GET /api/transactions/{id}
Возвращает информацию о конкретной транзакции по ID

**Параметры:**

| Параметр | Тип     | Обязательный | Описание                             |
| -------- | ------- | ------------ | ------------------------------------ |
| id       | Integer | да           | Уникальный идентификатор транзакции  |

**Ответ:**

| Код | Описание                               |
| --- | -------------------------------------- |
| 200 | Возвращает данные транзакции           |
| 404 | Транзакция с указанным ID не найдена   |

#### POST /api/transactions
Создаёт новую транзакцию

**Параметры:**

| Параметр        | Тип           | Обязательный | Описание                                  |
| --------------- | ------------- | ------------ | ----------------------------------------- |
| sellerId        | Integer       | да           | ID продавца (должен существовать)         |
| amount          | BigDecimal    | да           | Сумма транзакции (больше 0)               |
| paymentType     | String        | да           | Тип оплаты: CASH, CARD, TRANSFER          |
| transactionDate | LocalDateTime | да           | Дата и время транзакции в формате ISO 8601|

**Ответ:**

| Код | Описание                                      |
| --- | --------------------------------------------- |
| 201 | Транзакция успешно создана                    |
| 400 | Некорректные данные (отрицательная сумма и тд)|
| 404 | Продавец с указанным sellerId не найден       |

**Пример запроса:**
```json
{
  "sellerId": 1,
  "amount": 1500.50,
  "paymentType": "CARD",
  "transactionDate": "2026-05-15T14:30:00"
}
```

**Пример ответа (404):**
```json
{
  "message": "Entity not found",
  "errorMessage": "Not found Seller by id = 999",
  "errorTime": "2026-05-15T10:30:00"
}
```
#### PUT /api/transactions/{id}
Обновляет существующую транзакцию

**Параметры:**

| Параметр        | Тип           | Обязательный | Описание                                  |
| --------------- | ------------- | ------------ | ----------------------------------------- |
| id              | Integer       | да           | Уникальный идентификатор транзакции       |
| sellerId        | Integer       | да           | ID продавца                               |
| amount          | BigDecimal    | да           | Новая сумма транзакции (больше 0)         |
| paymentType     | String        | да           | Новый тип оплаты: CASH, CARD, TRANSFER    |
| transactionDate | LocalDateTime | да           | Новая дата и время транзакции             |

**Ответ:**

| Код | Описание                                      |
| --- | --------------------------------------------- |
| 200 | Транзакция успешно обновлена                  |
| 400 | Некорректные данные                           |
| 404 | Транзакция или продавец не найдены            |
